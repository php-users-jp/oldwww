PHP 4.3.3 Release Announcement

長期に渡るQAプロセス(訳注：Quality Assurance)を経て、 PHP 4.3.3がついにリリースされました。
このメンテナンスリリースでは、これまでにリリースされたPHPで発見された バグが多数解決され、同時にセキュリティの問題も修正されています。全ての ユーザに対して、出来る限り早い4.3.3へアップグレードが強く 推奨されます。

Bugfix Release

PHP 4.3.3では重要な修正、機能追加、改善が行われています。 その一部を以下にあげます：

  • Zend Engineを(可能な限り)POSIXソケットIOを使用するように改善
  • 潜在的なセキュリティ上の問題である整数/バッファオーバーフローの修正
  • multipart/form-data使用時に、文字の2バイト目に0x5cを含むファイルを 正しくアップロードできない問題の修正
  • each()で使用されるキーをバイナリセーフに修正
  • NSAPI SAPIの大幅な改善
  • IMAP拡張モジュールの改善
  • InterBase拡張モジュールの改善
  • iniファイルでDBAハンドラの'inifile'オプションをサポート
  • CLI, CGI版PHPにロングオプションを追加(e.g. --version)
  • preg_match*()関数にマッチを開始するオフセットを指定するための パラメータを追加
  • PHP同梱のExpatライブラリのバージョンを1.95.6に更新
  • PHP同梱のPCREライブラリのバージョンを4.3に更新
  • PHP同梱のGDライブラリのバージョンを2.0.15に更新
  • 100を超えるバグフィクス！

PHP 4.3.3における変更点の一覧についてはNEWSファイルを参照してください。
