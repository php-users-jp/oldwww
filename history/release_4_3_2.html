PHP 4.3.2 Release Announcement

長期に渡る(訳注：いつものことですが)QAプロセス(訳注：Quality Assurance)を経て、PHP 4.3.2がついにリリースされました。
このメンテナンスリリースでは以前のバージョンで発見された多くのバグが修 正されていますので、全てのPHPユーザに対してアップグレードが強く 推奨されます。

Bugfix Release

PHP 4.3.2では重要な修正、機能追加、改善が行われています。 その一部を以下にあげます：

  • 潜在的な危険があったintegerもしくはbufferオーバーフローの修正
  • いくつかの64bitプラットフォームでの問題の修正
  • 新しいApache 2.0 SAPIモジュール(sapi/apache2handler -> --with-apxs2 で有効になります)
  • session_regenerate_id()関数の追加(これは悪意あるセッション攻撃に 対して重要な働きをする関数で、有効なセッションのデータを 保ったままセッションIDを変更することができます)
  • dba拡張モジュールの改善
  • thttpd SAPIモジュールの改善
  • WindowsでのGDLIB version 1.x.x(php_gd.dll)のサポート終了
  • CLI版PHPのUnix用manページの追加
  • セキュリティ上の理由で、管理者が特定の名前を持ったクラスの使用を 禁止することができるdisable_classesオプションをphp.iniに追加
  • ...そして*多くの*バグフィックス

PHP 4.3.2における変更点の一覧についてはNEWSファイルを参照してください。
